# IMAP Connector v1.1.0 — Complete Follow-ups Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`).

**Goal:** Close every deferred item so the connector is feature-complete and host-ready: real XOAUTH2 (Gmail + Microsoft 365), real HTML→markdown, and an optional publishable HTTP layer (controller + routes + credential form) so the host wiring ships with the package. Finish with a flagship README.

**Architecture:** Build on the v1.0.0 package (`ImapConnector` extends `BaseConnector`; webklex behind `ImapClientInterface`; provider config under `connectors.providers.imap`). XOAUTH2 reuses the existing OAuth seam (`initiateOAuth`/`handleOAuthCallback`/`refreshTokenIfExpired`). The HTTP layer is OPT-IN (config toggle, default off) and publishable, so hosts that wire their own admin keep working unchanged.

**Tech Stack:** PHP 8.3–8.5, Laravel 12/13, webklex/php-imap ^6, league/html-to-markdown ^5, PHPUnit, Orchestra Testbench, Pint, PHPStan level 8 (`--memory-limit=512M`).

## Global Constraints

- `declare(strict_types=1);` everywhere. PSR-4 one-class-per-file under `src/`.
- Tests run via `vendor/bin/phpunit` (plain — Live self-skips). PHPStan: `vendor/bin/phpstan analyse --memory-limit=512M`. Pint: `vendor/bin/pint --test`.
- No secret (password, client_secret, access/refresh token) is ever logged, put in an exception message, audit metadata, the KB disk, or ingested metadata.
- XOAUTH2 providers: `google` (imap.gmail.com) and `microsoft` (outlook.office365.com). Client creds + URLs come from `config('connectors.providers.imap.xoauth2.<provider>')`. Per-installation `config_json.xoauth2_provider` selects which (default 'google'); `config_json.auth_mode = 'xoauth2'` activates the OAuth path.
- The vault stores xoauth2 `accessToken`/`refreshToken`/`expiresAt`; basic-auth stores the password as `accessToken` with null expiry. `makeClient()` for xoauth2 must use a FRESH (refreshed) access token.
- HTTP layer is OFF by default (`config('connectors.providers.imap.routes.enabled')`, default false). When enabled it registers routes; it is also publishable (`--tag=connector-imap-http`) for hosts that want to customize.
- Backward compatible: all v1.0.0 tests keep passing; basic-auth behavior unchanged.

```php
use League\HTMLToMarkdown\HtmlConverter;

private function htmlToMarkdown(string $html): string
{
    $converter = new HtmlConverter([
        'strip_tags' => true,           // drop unknown tags rather than keep raw
        'remove_nodes' => 'script style head',
        'hard_break' => true,
        'use_autolinks' => false,
    ]);

    try {
        return trim($converter->convert($html));
    } catch (\Throwable) {
        // Defensive: malformed HTML never breaks ingestion — fall back to text.
        return trim(html_entity_decode(strip_tags($html), ENT_QUOTES | ENT_HTML5, 'UTF-8'));
    }
}
```

`refreshTokenIfExpired`:
```php
public function refreshTokenIfExpired(int $installationId): ?string
{
    if ($this->authMode($installationId) !== 'xoauth2') {
        return $this->vault->getAccessToken($installationId); // basic: password, never expires
    }
    $access = $this->vault->getAccessToken($installationId);
    if ($access !== null) {
        return $access; // still valid
    }
    $refresh = $this->vault->getRefreshToken($installationId);
    if ($refresh === null) {
        return null;
    }
    $p = $this->xoauthProviderConfig($installationId);
    $resp = Http::asForm()->acceptJson()->post($p['token_url'], [
        'client_id' => $p['client_id'], 'client_secret' => $p['client_secret'],
        'grant_type' => 'refresh_token', 'refresh_token' => $refresh,
    ]);
    if (! $resp->successful()) {
        throw new ConnectorAuthException('XOAUTH2 token refresh failed: HTTP '.$resp->status());
    }
    $payload = $resp->json();
    $expiresAt = isset($payload['expires_in']) ? Carbon::now()->addSeconds((int) $payload['expires_in']) : null;
    $newRefresh = is_string($payload['refresh_token'] ?? null) ? $payload['refresh_token'] : $refresh;
    $this->vault->setCredentials($installationId, accessToken: (string) $payload['access_token'], refreshToken: $newRefresh, expiresAt: $expiresAt, extra: $this->vault->getExtra($installationId));
    $this->emitAudit('token_refreshed', installationId: $installationId);
    return (string) $payload['access_token'];
}
```

`makeClient`: for xoauth2 use `refreshTokenIfExpired()` as the secret instead of the raw stored access token:
```php
protected function makeClient(int $installationId): ImapClientInterface
{
    $secret = $this->authMode($installationId) === 'xoauth2'
        ? (string) ($this->refreshTokenIfExpired($installationId) ?? '')
        : (string) ($this->vault->getAccessToken($installationId) ?? '');
    return $this->makeClientWithPassword($installationId, $secret);
}
```

**Interfaces (routes, only when `routes.enabled`):**
- `GET  {prefix}/{installation}/credentials` → `credentials@form` (basic-auth: render the Blade form; xoauth2: redirect to `initiateOAuth`).
- `POST {prefix}/{installation}/credentials` → `credentials@store` → calls `connector->handleOAuthCallback($installation, $request)` (basic path: state + host/port/encryption/username/password fields).
- `GET  {prefix}/{installation}/oauth/callback` → `oauth@callback` → `connector->handleOAuthCallback(...)`.

**Design notes:**
- The controller resolves the connector from the registry/container (`app(ImapConnector::class)`), looks up the `ConnectorInstallation` (tenant-scoped), and delegates to the contract methods. It does NOT duplicate auth logic.
- `initiateOAuth` for basic returns the `credential_form_url`; the `form` action renders the Blade form posting to the `store` route with a `state` (issued via the connector). To keep CSRF: the form includes Laravel `@csrf` AND the connector `state` hidden field.
- Keep it minimal and host-overridable; everything publishable.
